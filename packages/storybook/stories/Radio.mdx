# RadioGroup and Radio Components

The `Radio` and `RadioGroup` components allow users to select a single option from a group. 
They are designed with accessibility and styling in mind, making them highly flexible and customizable.

### Properties

#### RadioGroup:
- **value** (`string | undefined`): The currently selected value in the group. Controls the state when used as a controlled component.
- **onChange** (`(value: string) => void`): Callback triggered when the selection changes.
- **children** (`React.ReactNode`): The `Radio` components to be displayed in the group.
- **customStyles** (`Partial<RadioGroupStyles>`): Object containing custom styles for specific elements of the `RadioGroup` and `Radio` components.

#### Radio:
- **value** (`string`): The value of the radio button, used to identify the selected option.
- **label** (`string`): The text displayed alongside the radio button.
- **disabled** (`boolean`): Disables the radio button, preventing interaction. Defaults to `false`.
- **className** (`string`): Additional CSS classes for the radio button.

Standalone usage without the need of a `RadioGroup` parent
- **onChange** (`(value: string) => void`): Callback triggered when the selection changes.
- **selectedValue** (`string`): Selected value per external state.
- **customStyles** (`Partial<RadioGroupStyles['isolatedRadio']>`): Object containing custom styles for specific elements of `Radio` used without a `RadioGroup` parent.
