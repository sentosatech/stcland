# Button Component

The `Button` component is a versatile UI element that can be used in various scenarios such as submitting forms, triggering actions, and more. It comes with several customization options such as:

### Properties

#### Base:
- **text** (`string`): Text label of the button. Defaults to `'No Text'`.
- **onClick** (`() => void`): Function to be called when the button is clicked. Defaults to a no-operation function.
- **leftIcon** (`React.ComponentType`): Optional icon component that can be rendered alongside the button text.
- **rightIcon** (`React.ComponentType`): Optional icon component that can be rendered alongside the button text.
#### Size Variants:
- **size** (`sm | md | lg`): Applies size variant. Defaults to `md`.
- **fullWidth** (`boolean`): If true, the button will take up the full width of its container. Defaults to `false`.
#### Color Variants:
- **highlightOnHover** (`boolean`): Changes background on hover when true.
- **type** (`[primary | secondary | tertiart]`): Applies the desired variant. Defaults to `primary`
- **disabled** (`boolean`): Disables the button when true. Defaults to `false`.
#### Shape Variants:
- **rounded** (`boolean`): Applies rounded corners to the button. Defaults to `true`.
#### Custom Styles:
- **className** (`string`): Additional CSS classes for the button.
- **customStyles** (`Partial<ButtonStyles>`): Allows passing custom styles for `root` and `button` directly.
#### To use when wrapped within a Form:
- **parentButtonProps** (`ParentButtonPropsShape`): Additional button properties like `type` and `form` for forms.

### WithStyles Wrapper

When a component is exported from the `@stcland/components/withStyles` module and wrapped in the `StcStylesProvider` it allow 
users to access styled versions of the core components. These components enhance the base functionality with added styling without
the need to pass a `customStyles` property as the **With Custom Styles** example above.
